# SteamWatch WinUI 手动验收清单

## 验收前准备

- 使用 `scripts\publish-portable.ps1 -Configuration Release -Platform x64` 生成绿色版。
- 从 `artifacts\publish\SteamWatch-win-x64` 启动 `SteamWatch.App.exe`。
- 确认本机已安装 Steam，并至少有一个可启动游戏。

## 基础启动

- 应用能正常启动并显示主窗口。
- 游戏页能读取 Steam 路径、当前用户和游戏列表。
- 游戏列表显示游戏名称、AppID、Steam 累计时长、运行状态。
- 关闭窗口行为默认最小化到托盘。
- 托盘菜单包含打开、暂停/恢复监控、设置、退出。

## 运行计时

- 启动一个 Steam 游戏后，游戏页状态在下一次轮询后变为运行中。
- 运行超过 1 分钟后，今日和本周分钟数增加。
- 退出游戏后，最终不足 1 分钟的会话尾数会补记 1 分钟。
- 重新启动应用后，今日和本周分钟数仍能从 `data\playtime.json` 恢复。

## 限额提醒

- 在游戏页选择一个游戏，保存每日仅提醒限额。
- 将限额设置为较小值，使当前使用进度达到 70%、85%、95% 或 100%。
- 达到阈值时弹出 Toast 提醒。
- 设置页关闭声音提醒后，后续提醒不播放声音。

## 强制退出

- 在游戏页选择一个游戏，保存每日强制退出限额。
- 达到限额后状态栏显示强退倒计时。
- 倒计时到期前，当前会话时长已保存。
- 倒计时到期后，应用先尝试温和关闭游戏窗口；失败时终止进程树。
- 全局强制退出规则会作用于当前所有运行中的 Steam 游戏。

## 统计页面

- 统计页显示每日明细。
- 统计页显示每周汇总，周周期为周一到周日。
- 运行中计时刷新后，统计页数据同步更新。

## 设置页面

- 保存关闭窗口行为为“退出应用”后，再关闭主窗口会退出应用。
- 保存关闭窗口行为为“最小化到托盘”后，再关闭主窗口会隐藏到托盘。
- 开机自启开关会同步当前用户 Run 注册表项。
- 强退倒计时秒数保存后，会影响后续强退倒计时。

## 绿色版产物

- `dist\SteamWatch-win-x64.zip` 可解压。
- 解压后能从目录内启动 `SteamWatch.App.exe`。
- 运行时数据写入应用目录下的 `data\`。
